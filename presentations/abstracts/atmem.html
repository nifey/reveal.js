ATMem: Adaptive Data Placement in Graph Applications on Heterogeneous Memories

¹College of William & Mary ²Lawrence Livermore National Laboratory

Heterogeneous Memory systems

  • High performance memory + High capacity memory
  • Eg: DRAM + NVM, MCDRAM + DRAM
  • Critical data to be placed on High performance memory
  • Tradeoff between performance and memory used
  • Graph applications
    • Massive data structures
    • Skewed access patterns

ATMem

  • Implemented as a Runtime library
  • Critical Data Chunk migration
    • Data structure granularity
    • Better utilization of memory
  • Lightweight sampling based profiling
    • Uses performance counters
    • Uses a Tree based clustering strategy
  • Multi-stage Multi-threaded migration
    • Data migration at the application level
    • Reduce TLB misses
    • Using Multiple threads to exploit high bandwidth

Evalutations

  • Evaluate on Real systems
    • NVM-DRAM testbed (Intel Xeon Platinum 8260L)
    • MCDRAM-DRAM testbed (Intel Xeon Phi 7200)
  • 5 graph applications: BFS, SSSP, BC, PR, CC
  • On 5 different graph datasets

Conclusion

  • ATMem selects 5%-18% data to be placed on high-performance memory
  • Compared to a baseline with all data in high capacity memory
    • achieves 1.7x-3.4x speedup on NVM-DRAM
    • achieves 1.2x-2.0x speedup on MCDRAM-DRAM
  • Compared to an ideal full DRAM system
    • achieves 9-54% slowdown on NVM-DRAM
    • while only using a fraction of DRAM
