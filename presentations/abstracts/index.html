Problem: Making reductions faster

  • Reductions are common operation in HPC
  • Eg: Sum, Min, Max, Logical ops, etc
  • Two types
    • Reduction over scalar types, Eg: Sum
    • Reduction over vector types, Eg: Histogram

Existing approaches

Privatization
  • Works well for small reduction variables
  • Cache pollution for large reduction variable
Atomics
  • Works well for larger reduction variables
  • Cache coherence invalidations and False sharing

Shortcomings of existing work

  • Doesn't work well for vector reduction variables
  • Modifies ISA => Libraries need to be modified
  • Modifies cache coherence protocol => Hard to verify

RiCH

  • Runtime assisted Hardware solution
  • Offloads reductions from CPU core to reduction module
  • Allows programmer to control cache pollution by annotating where to perform reduction

Advantages

  • Load modify store instructions performed together
  • Does not suffer from coherence effects
  • Updation at higher cache levels reduced cache pollution

Machine-agnostic and Communication aware Designs for MPI on Emerging Architectures

¹The Ohio State University

Problem

  • MPI runtimes use Static policies for Rank to core mapping
  • Allows specifying arbitrary mapping => Burden on Application Developer
  • "How can we redesign MPI runtimes to accurately construct physical to virtual hardware topologies for any given architecture?"

Proposed approach

  1. Construct an Architecture graph
    • Measures cache invalidation time across all threads (Benchmark based approach)
    • Only done once
  2. Construct the Communication graph
    • consider only two sided calls
    • Uses MPI_T interface
    • Measures frequency and volume of data communicated
  3. Do the Mapping taking both latency and memory requirement into account
